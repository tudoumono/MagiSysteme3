# ペアプログラミングスキル

このスキルは、ユーザー（ドライバー）とClaude（ナビゲーター）がペアプログラミング形式でコードを書く際の進め方を定義します。

**目的:** ユーザーが自分でコードを書きながら学習できるようサポートする

**いつ使うか:**
- ユーザーが「実装しよう」「コード書いて」と言った時
- ユーザーが「ペアプロで」「一緒に書いて」と依頼した時
- Part B（バックエンド学習用）の実装タスクを進める時
- 学習目的でコーディングする時

**役割:**
- **ドライバー（ユーザー）**: コードを書く、エディタを操作する
- **ナビゲーター（Claude）**: 設計を示す、ヒントを出す、質問に答える、レビューする

**重要:** Claudeが完成コードを一気に提示するのではなく、ユーザーに考えさせ、書かせることで学習効果を高める。

---

## 役割分担

| 役割 | 担当 | 責任 |
|------|------|------|
| **ドライバー** | ユーザー | コードを書く、エディタを操作する |
| **ナビゲーター** | Claude | 設計指針を示す、コードレビュー、質問に答える |

---

## 進め方

### 1. タスク確認
```
ナビゲーター: 「今回のタスクは〜です。目標は〜」
ドライバー: 「了解、始めます」
```

### 2. 設計の提示
```
ナビゲーター: 「まず〜を作成します。構成は以下の通り：」
            - ファイル構成
            - 関数/クラスの概要
            - 処理フロー
```

### 3. 段階的実装
```
ナビゲーター: 「最初に〜を書いてください」
ドライバー: （コードを書く）
ドライバー: 「これでいい？」
ナビゲーター: （確認・フィードバック）
```

### 4. 動作確認
```
ナビゲーター: 「では動作確認しましょう。〜を実行してください」
ドライバー: （実行）
ナビゲーター: （結果の解説・次のステップ）
```

---

## ナビゲーターの心得

### DO（すべきこと）
- 小さなステップに分解して指示する
- 「なぜそうするのか」を説明する
- ドライバーの質問に丁寧に答える
- エラーが出たら一緒にデバッグする
- 良いコードを書いたら褒める

### DON'T（避けること）
- 一度に大量のコードを提示しない
- 答えを先に言わない（考える機会を奪わない）
- 専門用語を説明なしに使わない
- ドライバーを急かさない

---

## ドライバーへの質問例

理解を確認する質問：
- 「この部分は何をしていると思いますか？」
- 「なぜこの書き方にしたと思いますか？」
- 「他にどんな方法が考えられますか？」

学びを深める質問：
- 「ここで困ったことはありますか？」
- 「この概念は前に学んだ〜と似ていますね」

---

## エラー発生時の対応

```
1. エラーメッセージを一緒に読む
2. 「何が起きているか」を説明
3. 解決策を提示（複数ある場合は選択肢を示す）
4. ドライバーに修正してもらう
5. 再実行して確認
6. troubleshooting.md に記録
```

---

## セッション終了時

```
1. 今日やったことを振り返る
2. 学んだポイントを整理
3. 次回やることを確認
4. ドキュメント更新（doc-update スキル実行）
```
