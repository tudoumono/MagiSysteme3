# ドキュメント更新スキル

このスキルは、MAGIシステムの学習ドキュメントを一貫した形式で更新するためのルールを定義します。

**目的:** ハンズオン形式で再学習できるドキュメントを維持する

**いつ使うか:**
- Step実装が完了し、動作確認に成功した後
- ユーザーが「ドキュメント更新して」「docに反映して」と依頼した時
- セッション終了前に進捗を記録する時
- 新しいSDK機能を使用する前（reference/strands_sdk.mdの更新）
- エラーを解決した後（reference/troubleshooting.mdの更新）

**重要:** このスキルに従わないと、学習記録が不完全になり、再学習時に困る。

---

## 更新対象ファイル

| ファイル | 役割 | 更新内容 |
|---------|------|---------|
| `docs/02_current_progress.md` | 進捗管理 | 完了マーク、次のタスク更新 |
| `docs/03_code_snapshot.md` | コード保存 | 動作確認済みコード全文 |
| `docs/handson/stepX.md` | ハンズオン教材 | 実装手順・つまずきポイント |
| `docs/reference/strands_sdk.md` | SDK仕様 | 新しく使用した機能を追記 |
| `docs/reference/troubleshooting.md` | エラー集 | 遭遇したエラーと解決策 |

---

## 更新フロー

```
動作確認成功
    ↓
1. handson/stepX.md 作成（実装手順・つまずきポイント）
    ↓
2. 02_current_progress.md 更新（完了マーク）
    ↓
3. 03_code_snapshot.md 更新（最新コード）
    ↓
4. reference/*.md 更新（必要に応じて）
```

---

## ハンズオン教材フォーマット

`docs/handson/stepX_xxxxx.md` のテンプレート：

```markdown
# Step X: タイトル

## 学習目標
- この Step で習得すること（箇条書き）

## 前提知識
- 必要な事前知識

## 実装手順

### 1. ファイル作成/準備
（何を作成するか、どこに配置するか）

### 2. コード実装
（ステップバイステップでコードを書く手順）

### 3. 動作確認
（実行コマンド、期待される出力）

## つまずきポイント
- よくあるエラーと対処法
- 間違えやすい点

## 学んだこと
- この Step で学んだ技術的なポイント

## チェックリスト
- [ ] 確認項目1
- [ ] 確認項目2
```

## 02_current_progress.md 更新ルール

### 完了マーク
```markdown
### X. タスク名 ✅

**目標:** 〜する → **完了！**
```

### 次のタスク更新
```markdown
## 次のタスク

### Y. タスク名 📋 ← 次はここ
```

## 03_code_snapshot.md 更新ルール

- 動作確認済みのコードのみ記載
- ファイルごとにセクション分け
- 学習メモを末尾に追加

```markdown
## agentcore/ファイル名.py

\`\`\`python
# 動作確認済みコード全文
\`\`\`

---

## 学習メモ
- ポイント1
- ポイント2
```

- 使用例コードを記載
